Enable Windows Features
Run two commands in PowerShell as Administrator

Press Windows key, type PowerShell, right-click → Run as Administrator. Then run both commands:

PowerShell (Admin) 
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
⚠️
Restart your PC after both commands complete. Do not skip the restart — the features won't activate without it.
  • Ran both dism commands successfully
  • Restarted PC
2
Enable SVM in BIOS
Turn on AMD virtualization — required for WSL2

Restart your PC and press Delete repeatedly as soon as it powers on to enter BIOS. (Some A520 boards use F2 — try Delete first.)

Once in BIOS, find the SVM setting using this table:

Brand Path to SVM / Virtualization
Gigabyte MIT → Advanced Frequency Settings → Advanced CPU Settings → SVM Mode → Enabled
MSI OC → CPU Features → SVM Mode → Enabled
ASRock Advanced → CPU Configuration → SVM Mode → Enabled
ASUS Advanced Mode (F7) → Advanced → CPU Configuration → SVM Mode → Enabled
Not sure Press F6 or look for a search bar, type SVM or Virtualization

Press F10 to Save & Exit, then let Windows boot normally.

  • Entered BIOS (Delete key on boot)
  • Found and set SVM Mode → Enabled
  • Saved with F10 and booted back into Windows
3
Install WSL2 + Ubuntu
Now that virtualization is on, WSL2 will install cleanly

Open PowerShell as Administrator again and run:

PowerShell (Admin) 
wsl --install

This downloads and installs Ubuntu automatically. When it finishes it will ask you to restart again. Do so.

After the restart, Ubuntu will open and ask you to create a username and password. Pick something simple — you'll use it occasionally.

💡
Verify it worked In PowerShell run: wsl --status — you should see "Default Version: 2" with no errors this time.
  • wsl --install ran without errors
  • Created Ubuntu username + password
  • wsl --status shows Default Version: 2, no errors
4
Install NVIDIA Drivers
Lets your RTX 4060 Ti work inside Docker containers
⚠️
Install the Windows driver only. Do NOT install CUDA separately inside WSL2 — the Windows driver handles it automatically.

1. Go to nvidia.com/Download/index.aspx

2. Select: Product Type = GeForce, Series = RTX 40, GPU = RTX 4060 Ti, OS = Windows 11

3. Download and install the latest driver.

4. After install, open Ubuntu (WSL2) and verify your GPU is visible:

Ubuntu (WSL2) Terminal 
nvidia-smi

You should see your RTX 4060 Ti listed with 8192MB VRAM. If you see it — great, move on.

  • Downloaded and installed latest NVIDIA driver
  • nvidia-smi in Ubuntu shows RTX 4060 Ti
5
Install Docker Desktop
The container runtime that runs the bot

1. Download Docker Desktop from docker.com/products/docker-desktop

2. During install, make sure "Use WSL 2 instead of Hyper-V" is checked ✅

3. After install, open Docker Desktop → Settings → Resources → WSL Integration → toggle on your Ubuntu distro ✅

4. Verify in Ubuntu terminal:

Ubuntu (WSL2) Terminal 
docker run --rm hello-world

You should see "Hello from Docker!" — that means Docker is working inside WSL2.

  • Docker Desktop installed
  • WSL Integration enabled for Ubuntu in Docker settings
  • docker run hello-world succeeds
6
Set Up the Bot
Extract files, add API keys, configure

Open your Ubuntu terminal. Copy the project from Windows into WSL2's filesystem (faster than running from /mnt/c/):

Ubuntu (WSL2) Terminal 
# Copy from your Windows Downloads folder into WSL2 home
cp /mnt/c/Users/$USER/Downloads/kronos-bot.tar.gz ~/
cd ~
tar -xzf kronos-bot.tar.gz
cd kronos-bot

# Create your .env file with Binance API keys
cp .env.example .env
nano .env

In the nano editor, replace the placeholder values:

.env file 
BINANCE_API_KEY=paste_your_key_here
BINANCE_API_SECRET=paste_your_secret_here

Press Ctrl+X, then Y, then Enter to save.

⚠️
Test on Binance Testnet first! Open config/config.json and set "testnet": true. Get free testnet keys from testnet.binance.vision. Run for a day before switching to real money.
  • Extracted kronos-bot into WSL2 home directory
  • Created .env with Binance API keys
  • Set "testnet": true in config/config.json for initial testing
7
Build & Launch
First build takes ~15 min — downloads Kronos AI model
Ubuntu (WSL2) Terminal 
# Build the Docker image (first time only — downloads ~4GB)
docker compose build

# Start the bot in background
docker compose up -d

# Watch logs to confirm it started
docker compose logs -f

# Confirm GPU is being used
docker compose logs | grep -i "cuda\|gpu\|device"
✅
Dashboard is live at http://localhost:8080 Open this in your Windows browser once the bot starts.
  • docker compose build completed
  • docker compose up -d started successfully
  • Dashboard visible at http://localhost:8080
  • Logs show CUDA device being used
8
Go Live
Switch from testnet to real trading once you're satisfied
Ubuntu (WSL2) Terminal 
# Edit config — flip testnet to false, update .env with real keys
nano config/config.json   # set "testnet": false
nano .env                 # swap in your real Binance API keys

# Restart bot with new config
docker compose down
docker compose up -d
